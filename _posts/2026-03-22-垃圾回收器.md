## 如何判断垃圾

如何判断堆中的对象是否为垃圾？直观想法是判断是否有引用指向它。因此有两种办法：**引用计数法**和**可达性分析**。目前 **可达性分析** 更常用。

### 引用计数法

- 每个对象分配一个计数器，表示引用数目
- 新引用指向对象时计数器 +1，引用失效时计数器 -1
- **问题：循环引用**
  - A 对象引用 B，对象 B 又引用 A
  - 即使没有其他引用，A 和 B 也不会被释放

### 可达性分析

- 从 **GC ROOTS** 出发，引用关系视为边
- 能到达的对象是存活的，不能到达的是垃圾
- **GC ROOTS** 可能是：
  1. 栈中的引用
  2. 本地方法栈中的引用
  3. 常量引用
  4. 静态变量引用

#### 三色标记法

- 白色：未访问对象（可能是垃圾）
- 灰色：已访问，但引用对象未全部访问
- 黑色：对象及其引用已访问完成

**流程**：

1. 所有节点初始化为白色
2. 将 GC ROOTS 节点标灰
3. 循环处理灰色节点：引用标灰，本身标黑
4. 循环结束后，白色节点即为垃圾

## 垃圾回收算法

常见算法：

1. **标记-清理（Mark-Sweep）**
2. **复制（Copying）**
3. **标记-整理（Mark-Compact）**
4. **分代回收（Generational GC）**

### 标记-清理算法

- 先标记垃圾，再清理
- **缺点**：
  - 内存碎片
  - 效率低

### 复制算法

- 内存分两块，每次只使用一块
- 当满时，将存活对象复制到另一块，再清理原块
- **优点**：避免碎片
- **缺点**：
  - 实际可用内存减半
  - 对象大时复制慢

### 标记-整理算法

- 标记垃圾，移动存活对象到一边，清理空闲区
- **缺点**：整理成本高

### 分代回收算法

- 新生代：短生命周期，复制算法
- 老年代：长生命周期，标记-清理/整理
- **优点**：
  - 减少扫描范围
  - 可针对不同代使用不同算法

## 垃圾回收器

| 垃圾回收器 | 新生代算法 | 老年代算法 | 特点 |
|------------|------------|------------|------|
| Serial     | 标记-复制  | 标记-整理  | 单线程, STW |
| ParNew     | 标记-复制  | -          | Serial 多线程版本 |
| Parallel Scavenge | 标记-复制 | 标记-整理 | 注重吞吐量 |
| CMS        | 标记-清理  | -          | 并发回收, 停顿短 |
| G1         | 复制+标记整理 | -        | 可预测停顿, Region 管理 |

### Serial

- 单线程，STW
- 新生代：标记-复制
- 老年代：标记-整理

### ParNew

- Serial 多线程版本
- **Par**：并行（多线程）
- **Concurrent**：并发（用户线程不中断 GC）

### Parallel Scavenge + Parallel Old

- 新生代：标记-复制
- 老年代：标记-整理
- 注重吞吐量

### CMS

- 并发回收器，目标最短停顿时间
- **步骤**：
  1. 初始标记（STW）
  2. 并发标记
  3. 重新标记（STW 修正）
  4. 并发清理
- **缺点**：内存碎片、浮动垃圾

### G1（Garbage First）

- 目标：可预测停顿 + 高吞吐量 + 避免碎片
- **改进点**：
  1. 逻辑分代（Region）而非物理分代
  2. 优先回收垃圾最多的 Region
  3. 预测停顿时间，动态选择 Region
  4. 底层复制算法，宏观标记-整理
  5. 避免 CMS 浮动垃圾

**G1 回收流程**：

1. 初始标记（STW）
2. 并发标记
3. 最终标记（STW）
4. 筛选回收高价值 Region
