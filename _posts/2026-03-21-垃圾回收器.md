# 垃圾回收器总结

垃圾回收器一直是比较重要的问题，无脑的死记硬背不会解决任何问题。但是垃圾回收器这块内容又比较杂，因此写下这篇 blog，主要帮自己梳理一下垃圾回收器的一些复杂问题。

核心可以分为三部分：

1. 如何判断垃圾
2. 垃圾回收的算法
3. 垃圾回收器

这三个概念，就相当于一个市的领导要清扫垃圾：

- 第一步：判断垃圾 → 明确哪些是垃圾（落叶算不算？）
- 第二步：垃圾回收算法 → 知道怎么扫
- 第三步：垃圾回收器 → 具体执行任务的队伍（效率有高有低）

------

## 一、如何判断垃圾

如何判断堆中存放的对象是否为垃圾？

直观想法：判断是否还有引用指向它。

因此产生了两种方法：

- 引用计数法
- 可达性分析（主流）

------

### 1. 引用计数法

对于每一个对象：

- 分配一个计数器
- 有引用 → +1
- 引用失效 → -1

问题：

如果：

- A 引用 B
- B 引用 A

那么它们的引用计数都不会变成 0 → **无法回收**

------

### 2. 可达性分析（主流）

从 **GC Roots** 出发：

- 引用关系 = 边
- 能到达的对象 → 存活
- 不能到达的对象 → 垃圾

#### GC Roots 包括：

1. 栈中的引用
2. 本地方法栈中的引用
3. 常量的引用
4. 静态变量的引用

------

### 三色标记法

对象分为三种颜色：

- **黑色**：已访问，且引用已处理完成
- **灰色**：已访问，但引用未处理完
- **白色**：未访问（可能是垃圾）

#### 执行流程：

1. 所有节点设为白色
2. GC Roots 设为灰色
3. 遍历灰色节点：
   - 引用对象设为灰色
   - 自身设为黑色
4. 重复，最终剩下白色对象 → 清除

------

## 二、垃圾回收算法

第一步解决“谁是垃圾”，接下来解决“怎么回收”。

常见算法：

1. 标记-清理
2. 复制
3. 标记-整理
4. 分代回收

------

### 1. 标记-清理算法

流程：

- 标记垃圾
- 统一清理

问题：

1. 内存碎片
2. 效率不高

------

### 2. 复制算法

思路：

- 内存分成两块
- 每次只用一块
- 存活对象复制到另一块

优点：

- 无内存碎片

缺点：

1. 内存利用率低（只能用一半）
2. 大对象复制成本高

------

### 3. 标记-整理算法

流程：

- 标记垃圾
- 存活对象移动到一侧
- 清理边界外空间

特点：

- 无碎片
- 但移动成本高 → 效率不高

------

### 4. 分代回收算法

背景：Java 堆分为：

- 新生代
- 老年代

特点：

- 新生代：对象“朝生暮死”
- 老年代：对象存活时间长

策略：

- 新生代 → 复制算法（成本低）
- 老年代 → 标记-清除 / 标记-整理

------

### 为什么要分代？

1. **减少扫描范围**
    → 不需要每次全堆扫描
2. **不同区域用不同算法**
    → 提高效率

------

## 三、垃圾回收器

重点考点：**CMS 和 G1**

主要回收器：

- Serial
- ParNew
- Parallel Scavenge
- Serial Old
- Parallel Old
- CMS
- G1

------

### 1. Serial

特点：

- 单线程
- 必须 STW（Stop The World）

算法：

- 新生代 → 标记-复制
- 老年代 → 标记-整理

------

### 2. ParNew

- Serial 的多线程版本

注意：

- Parallel（并行）：多线程执行 GC
- Concurrent（并发）：GC 与用户线程同时执行

------

### 3. Parallel Scavenge + Parallel Old

- Parallel Scavenge → 标记-复制
- Parallel Old → 标记-整理

目标：

👉 **吞吐量优先**

------

### 4. CMS（Concurrent Mark Sweep）

特点：

- 第一个并发垃圾回收器
- 目标：**最短停顿时间**
- 算法：标记-清除

#### 执行步骤：

1. 初始标记（STW）
2. 并发标记
3. 重新标记（STW）
4. 并发清除

#### 缺点：

1. 内存碎片
2. 浮动垃圾

------

### 5. G1（Garbage First）

目标：

👉 在 **可预测停顿时间** 下：

- 提高吞吐量
- 避免碎片

------

#### 执行步骤：

1. 初始标记（STW）
2. 并发标记
3. 最终标记（STW）
4. 筛选回收

------

#### 核心改进：

1. Region 设计（逻辑分代）
   - 不再物理分代
   - 划分多个 Region
2. 优先级回收
   - 维护优先队列
   - 优先回收“垃圾最多”的区域
3. 可预测停顿时间模型
   - 设定回收时间上限
   - 动态选择回收多少 Region
4. **避免碎片**
5. **解决 CMS 浮动垃圾问题**
6. 算法本质
   - 局部：复制算法
   - 整体：标记-整理
